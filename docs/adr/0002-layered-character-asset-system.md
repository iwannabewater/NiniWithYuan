# Separate character identity from surface-specific assets

Each protagonist will be defined once through a shared character bible, then expressed through separate UI illustration, gameplay pose-atlas, and brand-icon asset families. The game uses purpose-built poses for locomotion, airborne states, skills, attacks, damage, recovery, and celebration instead of shrinking or warping full-body UI illustrations. A motion resolver adds state-specific bob, lean, lift, squash, stretch, landing settle, and artifact trails around those poses, preserving detailed silhouettes without adding a skeletal-animation runtime to the dependency-light Android WebView build. Directional production is hybrid: generic idle, run, and fall poses may be mirrored, while turns, takeoff and landing, artifact handling, skills, attacks, and damage prefer authored left and right variants so costume construction remains coherent. If a generated directional cell is visibly invalid or contaminated, the atlas manifest may mark the clean opposite-facing source pose with `mirror: true`; the runtime otherwise never mirrors an authored directional pose. State timing remains explicit in the atlas contract so future multi-frame loops can be added without changing the resolver or gameplay contracts.
